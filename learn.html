How your Crypto Dashboard actually works

A visual, beginner-friendly breakdown of every piece of the project you built — from APIs to automation.

00 — The big picture

Your project is a data pipeline

Think of your dashboard like a newspaper. Every hour, a reporter (your Python script) goes out, gathers the latest crypto prices, asks an analyst (AI) for their opinion, writes it all down (JSON files), and the printing press (your HTML page) turns it into something people can read.

Real-world analogy
Reporter = Python script  •  Notebook = JSON files  •  Newspaper = HTML page  •  Alarm clock = GitHub Actions

That's the whole project in one picture. Now let's break down each piece.

01 — Foundation concept

What is an API?

API stands for Application Programming Interface. That sounds intimidating, but it's just a waiter at a restaurant.

Analogy
You (the customer) don't walk into the kitchen and cook your own food. Instead, you tell the waiter (the API) what you want. The waiter goes to the kitchen (the server), gets your food (the data), and brings it back to your table. You never need to know how the kitchen works.

Your project uses three APIs:

CoinGecko API
A free service that gives you live crypto prices, 24h price changes, and market caps for BTC, ETH, and SUI. No key needed.
TradingView API
Provides technical analysis data for Gold (XAU) — price, RSI indicator, and support/resistance levels from professional traders.
Groq API
A super-fast AI service running Meta's Llama model. Your script sends market data and asks it to write a trade analysis. Requires an API key.
API Key
Like a membership card. Some APIs are free to use (CoinGecko), others need a key to identify you (Groq). It's a long secret text string you keep private.
02 — The brain

The Python script: update_dashboard.py

Python is a programming language — a way to write instructions that a computer can follow. Your script is like a recipe that tells the computer exactly what to do, step by step.

Analogy
Python is like writing a cooking recipe. Each line is one instruction: "first, crack 2 eggs" becomes "first, fetch Bitcoin's price." The computer reads from top to bottom and does exactly what you wrote.

What happens when the script runs

1

Fetch crypto prices

Calls CoinGecko's API to get live prices, 24h changes, and market caps for Bitcoin, Ethereum, and SUI.

2

Fetch price history (OHLC)

Gets 14 days of candlestick data (Open, High, Low, Close) for each crypto. This is needed to calculate RSI.

3

Calculate RSI

RSI (Relative Strength Index) is a number from 0-100 that tells you if something is overbought (above 70) or oversold (below 30). The script calculates this from the price history.

4

Ask AI for analysis

Sends all the data (price, change, RSI, market cap) to Groq's Llama AI model with a carefully written prompt. The AI returns a JSON object with Elliott Wave analysis, momentum reading, key levels, trend bias, trade plan, and risk notes.

5

Fetch Gold data separately

Gold uses TradingView instead of CoinGecko — it's a commodity, not a crypto. The script tries multiple exchanges until one works.

6

Save everything as JSON files

Writes four files: btc.json, eth.json, sui.json, xau.json — each containing price, analysis, and metadata.

Key building blocks in the code

import
Loads a pre-built tool someone else made. Like borrowing a power drill instead of building one. import requests gives you the ability to call APIs.
function (def)
A reusable block of code. def fetch_crypto_prices(): defines a set of steps you can run anytime by calling its name.
variable
A named container for data. price = 70984 stores the number 70984 in a box labelled "price" that you can use later.
dictionary (dict)
A collection of key-value pairs, like a real dictionary. {"symbol": "BTC", "price": 70984} — you look up "symbol" and get "BTC".
requests.get()
Sends a "GET" request to an API — like asking the waiter "what's on the menu?" It returns the data from the server.
requests.post()
Sends data TO an API — like handing the waiter your order. Used for the Groq AI call because you're sending the prompt along with it.

A taste of the actual code

# This is a comment — the computer ignores it, it's just a note for humans

def fetch_crypto_prices():
    # Build the URL with the coins we want
    url = "https://api.coingecko.com/api/v3/simple/price?ids=bitcoin,ethereum,sui..."

    # Ask CoinGecko for the data (like visiting a website)
    r = requests.get(url)

    # Convert the response into Python data we can work with
    return r.json()

Every function in your script follows this pattern: build a URL or request, send it, get data back, do something with it.

03 — The data format

What is JSON?

JSON (JavaScript Object Notation) is just a way to organize data as text. Think of it as a form you fill out — it has labels (keys) and values.

Analogy
JSON is like a business card. It has structured information — Name: Bitcoin, Price: $70,984, Change: +3.87%. Both humans and computers can read it easily.
{
  "symbol": "BTC",              // The label (key) and value
  "price": 70984,               // Numbers don't need quotes
  "change_24h": 3.87,           // Decimal numbers work too
  "analysis": {                // You can nest objects inside objects
    "trend_bias": "bullish",
    "key_levels": {
      "support": ["$68,000", "$66,000"],  // Arrays = lists of items
      "resistance": ["$72,000", "$74,000"]
    }
  }
}

Why JSON? Because it's the universal language between systems. Your Python script writes JSON, and your HTML page reads JSON. They don't need to know anything about each other — they just agree on the format. This is how most of the internet works.

04 — The display

The HTML frontend: index.html

HTML (HyperText Markup Language) is the language every web page is written in. If the Python script is the brain, HTML is the face — it's what users actually see and interact with.

Your index.html is made of three languages working together:

HTML
The skeleton. Defines what's on the page — headings, buttons, containers. Like the frame of a house.
CSS
The paint and decoration. Defines how things look — colors, fonts, spacing, the dark terminal style. Like interior design.
JavaScript (JS)
The electricity. Makes things interactive — switching tabs, loading data, updating numbers. Makes the house "smart."
fetch()
JavaScript's way of loading a file. When you open the page, JS uses fetch('data/btc.json') to read the JSON file and display it.

How the page loads data

When someone opens index.html in a browser, the JavaScript at the bottom of the file runs automatically. Here's what it does:

1

Page opens in browser

The browser reads the HTML (structure), applies CSS (styling), then runs the JavaScript code.

2

JavaScript calls loadAsset('btc')

This fetches data/btc.json, reads the JSON data, and passes it to the rendering function.

3

renderPanel() builds the display

Takes the raw data and creates HTML elements — the price, change percentage, RSI bar, key levels, trade plan, and risk note.

4

Other assets load with a delay

ETH loads after 500ms, SUI after 1000ms, XAU after 1500ms. This avoids hitting the server too hard all at once (a technique called staggering).

05 — The automation

GitHub Actions: the robot worker

Let's break this into two concepts: Git/GitHub and GitHub Actions.

Git & GitHub

Analogy
Git is like a magical notebook that remembers every version of everything you've ever written. Made a mistake? Go back to yesterday's version. GitHub is like Google Drive for that notebook — it stores it online so you (and others) can access it anywhere.
Repository (repo)
Your project folder on GitHub. Contains all your files and their entire history.
Commit
A "save point." When you commit, Git takes a snapshot of all your files at that moment. You can always go back to any commit.
Push
Upload your local commits to GitHub so they're stored online and others can see them.
Secrets
Private values (like your Groq API key) stored safely in GitHub. Your code can use them, but nobody can see them.

GitHub Actions

GitHub Actions is a free automation service built into GitHub. You write a YAML file (a simple config format) that says "do this task on this schedule," and GitHub's servers do it for you — no computer of yours needs to be running.

The update.yml file is the instruction sheet. It tells GitHub: "Every hour on the hour, spin up a fresh computer, install Python, run my script, and if there's new data, save it back to the repository." The whole process takes about 30-60 seconds and costs you nothing.

06 — The intelligence

How the AI analysis works

This is the part closest to your heart — and to your future AGI ambitions. Here's what's actually happening when your script "asks AI for analysis."

Analogy
Imagine you're a financial reporter. You have the raw numbers (price, RSI, market cap). You walk over to an analyst's desk (the AI model), hand them a briefing sheet (the prompt), and ask them to write a structured report. They hand you back a filled-in form (the JSON response).

The prompt engineering

Your script builds a prompt — a carefully written instruction telling the AI exactly what data it has and exactly what format to respond in. This is a critical skill called prompt engineering, and you already have 2 years of experience with it as a Tier 1 user.

# What your script sends to Groq's Llama model:

"You are a professional crypto market analyst.
 Analyze BTC/USD and return ONLY a valid JSON object.

 Market data:
 - Price: $70,984.00
 - 24h Change: 3.87%
 - Market Cap: $1,419,742,320,932
 - RSI (14): 65.83 (neutral)

 Return this exact JSON structure:
 { elliott_wave, momentum, key_levels, trend_bias,
   trade_plan, risk_note }"

The AI stack

1

Groq (the platform)

Groq is a company that runs AI models on special hardware (LPUs) that makes them extremely fast. Think of it as a super-fast restaurant kitchen — same recipes, faster cooking.

2

Llama 3.1 8B (the model)

This is Meta's open-source AI model. "8B" means 8 billion parameters — think of parameters as the model's "brain cells." It's not the biggest model, but it's fast and good enough for structured analysis. For context, GPT-4 has ~1.8 trillion parameters.

3

Temperature: 0.3

This controls creativity. 0.0 = completely predictable/factual. 1.0 = very creative/random. Your script uses 0.3 — mostly factual with a tiny bit of creative interpretation. Perfect for market analysis.

4

The response

The AI returns a JSON object with exactly the fields you asked for. Your script parses it (reads the structured data) and saves it into the asset's JSON file.

Key takeaway for your AGI journey: This is a simple example of what AI engineers call an "AI pipeline" — data goes in, AI processes it, structured output comes out. More complex systems chain multiple AI calls together, add memory, and let the AI use tools. That's the path from here to building agents and eventually more autonomous systems.

07 — The full loop

How it all comes together

Every single hour, this entire sequence plays out automatically — without you doing a thing:

1

GitHub Actions wakes up

At the top of every hour, GitHub's servers fire up a fresh virtual machine.

2

Python script runs

Calls CoinGecko for crypto prices, TradingView for gold, calculates RSI, then asks Groq AI to analyze everything.

3

JSON files are updated

Four fresh data files are written with the latest prices and analysis.

4

GitHub commits & pushes

The new data is saved to the repository with a timestamp.

5

Dashboard refreshes

Next time anyone opens index.html, they see the latest data. No manual work needed.
